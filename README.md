# Инструкция по запуску проекта на Windows

Следуйте этим шагам, чтобы развернуть и запустить проект на локальной машине под управлением Windows.

## 1. Предварительные требования

Перед началом убедитесь, что у вас установлены:

1.  **Node.js** (версия 18 или выше).
    *   Скачать: [https://nodejs.org/](https://nodejs.org/) (рекомендуется версия LTS).
    *   При установке убедитесь, что стоят галочки для установки `npm` и добавления в `PATH`.
2.  **Git** (для клонирования или работы с версиями).
    *   Скачать: [https://git-scm.com/download/win](https://git-scm.com/download/win)

## 2. Установка зависимостей

1.  Откройте терминал (PowerShell или командную строку `cmd`) в папке с проектом.
2.  Выполните команду для установки всех библиотек:
    ```powershell
    npm install
    ```
    *Эта команда создаст папку `node_modules`, которой изначально не было в архиве.*

## 3. Настройка базы данных (SQLite)

Проект использует локальную базу данных SQLite. Вам нужно применить структуру базы данных и заполнить её начальными данными.

1.  Создайте файл базы данных и примените миграции:
    ```powershell
    npx prisma migrate deploy
    ```
    *Если команда завершится успешно, вы увидите сообщение о том, что миграции применены.*

2.  Сгенерируйте клиент Prisma (необязательно, если предыдущий шаг сделал это, но рекомендуется):
    ```powershell
    npx prisma generate
    ```

3.  Заполните базу данных тестовыми аккаунтами (Админ, Учитель, Студент):
    ```powershell
    npx tsx scripts/seed.ts
    ```
    *При первом запуске `npx` может спросить разрешение на установку пакета `tsx` — нажмите `y` (yes) и Enter.*

## 4. Запуск проекта

1.  Запустите режим разработки:
    ```powershell
    npm run dev
    ```
2.  Откройте браузер и перейдите по адресу: [http://localhost:3000](http://localhost:3000)

## 5. Важное примечание: Сканер QR-кодов

⚠️ **Внимание:** Функционал сканирования QR-кодов требует доступа к камере устройства. Браузеры блокируют доступ к камере на сайтах без защищенного соединения (HTTPS), за исключением `localhost`.

Однако, если вы попытаетесь открыть сайт с мобильного телефона в той же сети (по IP-адресу компьютера), камера **не будет работать**, так как соединение будет считаться небезопасным (HTTP).

**Чтобы сканер QR-кодов заработал полноценно (например, с телефона), проект необходимо:**
*   Либо опубликовать на хостинге с поддержкой HTTPS (Vercel, VPS и т.д.).
*   Либо использовать туннелирование для локальной разработки (например, через `ngrok` или `localtunnel`), которое предоставит временную ссылку с HTTPS.
